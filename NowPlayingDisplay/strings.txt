PLUGIN_NOWPLAYINGDISPLAY_VIZ
	EN	Audio visualizer (experimental)

PLUGIN_NOWPLAYINGDISPLAY_VIZ_DESC
	EN	Adds a Visualizer display mode that reacts to the actual music. To do this the display quietly streams the current track to itself (muted) and analyses it. It does not affect your players, register a player, or change playback — it only listens. Note: it uses some network bandwidth and asks the server to transcode the current track to MP3, which is a small extra load on the server. Leave off to keep the display a passive viewer.

PLUGIN_NOWPLAYINGDISPLAY_VIZ_LABEL
	EN	Enable the Visualizer mode

PLUGIN_NOWPLAYINGDISPLAY_VIZ_DELAY
	EN	Default visualizer sync offset (ms)

PLUGIN_NOWPLAYINGDISPLAY_VIZ_DELAY_DESC
	EN	Fallback offset used when a player has no per-player offset set above. Positive pushes the visualization earlier (use if it lags the sound), negative later. Range -2000 to 2000. Note: very low bass naturally resolves a little later than treble in any FFT analyzer — that part isn't corrected by this offset.

PLUGIN_NOWPLAYINGDISPLAY_VIZ_PRESETS
	EN	Visualizer delay presets

PLUGIN_NOWPLAYINGDISPLAY_VIZ_PRESETS_DESC
	EN	Define named sync-offset presets (in milliseconds) below, then pick one from the "Visualizer offset" dropdown next to each player above. This lets each room or display device keep its own dialled-in delay. First time: add a preset and Save — the new preset then appears in every player's dropdown, so choose one and Save again. Positive = visualization earlier, negative = later. Leave a row blank to remove that preset.

PLUGIN_NOWPLAYINGDISPLAY_VIZ_SERVER
	EN	Server-side audio analysis (experimental)

PLUGIN_NOWPLAYINGDISPLAY_VIZ_SERVER_DESC
	EN	Analyse the audio on the server (via a headless SqueezeLite + CAVA + WebSocket bridge) and render the visualizer from the server's band data, bypassing the Web Audio API. This is what allows the visualizer to work in Safari, iOS, and smart-TV browsers (which can't analyse audio reliably client-side). Requires the bridge helper to be running. Leave OFF for normal Chrome/Edge use.

PLUGIN_NOWPLAYINGDISPLAY_VIZ_SERVER_ENABLE
	EN	Use server-side analysis when a bridge URL is configured

PLUGIN_NOWPLAYINGDISPLAY_VIZ_SERVER_URL_DESC
	EN	WebSocket URL of the running visualizer bridge (e.g. ws://lms-host:8770/). Used by the page to receive band data. The display device must be able to reach this URL.

PLUGIN_NOWPLAYINGDISPLAY_VIZ_PLAYER_MAC_DESC
	EN	MAC address of the dedicated headless Visualizer SqueezeLite player (the one started with -m &lt;mac&gt; -v). This is the ONLY player the plugin will command in server-side mode — a strict safety guard prevents commands ever reaching real listening players.

PLUGIN_NOWPLAYINGDISPLAY_VIZ_AUTOFOLLOW
	EN	Auto-follow the room (mirror)

PLUGIN_NOWPLAYINGDISPLAY_VIZ_AUTOFOLLOW_DESC
	EN	On (default): the plugin mirrors the active/selected room onto the Visualizer player. Turn OFF only to experiment with native LMS sync grouping — when off the plugin sends no playlist/seek commands to the Visualizer, so you can manually sync it into a room's group (as a follower) without the mirror collapsing the shared queue. The Visualizer player and FFT helper keep running, so audio capture still works.

PLUGIN_NOWPLAYINGDISPLAY_VIZ_HELPER_AUTORUN
	EN	Auto-start bundled FFT helper when server mode is on

PLUGIN_NOWPLAYINGDISPLAY_VIZ_HELPER_AUTORUN_DESC
	EN	The plugin can manage the bundled Python FFT helper (Bin/npd-vizfft.py) directly — spawning it when server mode is enabled, restarting it if it dies, and stopping it when server mode is turned off. Disable only if you run the helper externally yourself (e.g. as a separate service).

PLUGIN_NOWPLAYINGDISPLAY_VIZ_SQZ_AUTORUN
	EN	Auto-start dedicated Visualizer SqueezeLite when server mode is on

PLUGIN_NOWPLAYINGDISPLAY_VIZ_SQZ_AUTORUN_DESC
	EN	The plugin can also manage the dedicated headless SqueezeLite instance that feeds the FFT helper via shared memory. Same lifecycle as the helper. Disable only if you already run it externally (e.g. as a systemd unit you wrote yourself).
